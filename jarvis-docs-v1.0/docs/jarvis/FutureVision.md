# Jarvis Future Vision

**Horizon 1 (this year):** Jarvis is the reason a solo-founder-plus-tiny-team operates with the institutional memory and decision discipline of a 50-person company — while remaining invisible: a repo, a brief, a queue of well-framed decisions.

**Horizon 2 (multi-startup):** the Startup Factory turns company creation into a repeatable method: diligence with confidence labels, ontology-first design, gated builds, accuracy-style proof obligations — each new venture inherits the discipline on day 1 and the platform code on day 30. The portfolio Executive view allocates founder attention and capital as explicit Type 1 decisions.

**Horizon 3 (self-improving):** capabilities are reviewed like agents — KPIs, evals, promotion/demotion — so Jarvis improves by the same loop the product uses on comps. The end state is not "AI runs the company"; it is that *judgment is the only thing the founder still has to supply*, and everything judgment needs — context, options, evidence, history — is one query away.

**What Jarvis will never become (standing boundaries):** a product we sell (until a deliberate ADR says otherwise) · a system that touches customer tenant data · a personality theater of 50 agents · an excuse to defer the actual company. The measure of Jarvis, permanently: the startups it operates ship faster, decide better, and forget nothing.
