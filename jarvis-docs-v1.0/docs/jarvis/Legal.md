# Legal Capability

**Purpose:** the company's obligations, exposures, and documents — tracked, current, and surfaced before they bite. Counsel remains counsel; Jarvis is the memory and the clock.

**Responsibilities:** **obligation register** (every contract's key terms: renewal/termination dates, data-use rights — vendor data derivative/persistence clauses get first-class fields per DD gate 4 — SLAs, exclusivities) with alerting · policy library (ToS, privacy policy, DPAs — versioned, review-dated) · compliance calendar (SOC 2 milestones, privacy-law effective dates from ResearchSystem alerts, 10DLC attestations) · contract intake SOP (new contract → extraction → counsel checklist → register) · risk register (legal section, linked to DOC-110 L1–L8) · licensing hygiene (OSS table enforcement partner to Engineering).
**Inputs:** contracts, counsel memos, regulatory alerts. **Outputs:** register, alerts, counsel-ready summaries, redline first-pass notes (L1 drafts, never advice). **Memory writes:** everything — legal memory is precisely the memory that must never live in one person's inbox.
**Approval rules:** L1 ceiling structurally: no signing, no filing, no advice; anything ambiguous routes to counsel with an assembled context pack (which is the actual time-saver).
**Failure modes:** extraction misses a poison clause → intake SOP requires counsel confirmation on flagged clause types (auto-flag list: indemnity, data use, exclusivity, auto-renewal, liability caps); register rot → renewal alerts are hook-scheduled, not goodwill-scheduled.
**KPIs:** obligations with owners + dates: 100% · missed renewals/deadlines: 0 · counsel prep time per matter (→ down) · days from regulatory alert → assessed impact.
