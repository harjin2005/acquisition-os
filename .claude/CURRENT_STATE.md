## Fixed this session (foundation bugs, not new features)
1. `backend/app/modules/admin/models.py`: `AuditEntry.metadata` column renamed to `details` —
   `metadata` is a reserved attribute name in SQLAlchemy's Declarative API and crashed
   `alembic upgrade head` on every fresh install (import chain: `db/registry.py` imports all
   module models for Alembic autogenerate, including `admin`). No migration exists yet for
   this table, so no DB schema was affected — pure Python-level fix.
2. `backend/requirements.txt`: added `email-validator==2.2.0`. `pydantic.EmailStr` (used in
   `app/modules/identity/schemas.py`) requires it at runtime; it was missing, so a fresh
   `pip install -r requirements.txt` produced a backend that couldn't boot.
3. `frontend/yarn.lock` was missing from the repo despite CI's `ci.yml` already referencing
   `cache-dependency-path: frontend/yarn.lock`. Generated by `yarn install` this session —
   should be committed so CI's cache step and reproducible installs actually work.
