# How to use these session cards (READ THIS FIRST)

You (the founder) do NOT write prompts anymore. You copy-paste a card.

## The simple daily routine
1. Open a new Claude Code session.
2. Decide which ROLE you need right now (usually just one at a time).
3. Open that role's card in this folder. Copy the whole "PROMPT TO PASTE" block.
4. Paste it into Claude Code. Press enter.
5. The session reads CLAUDE.md, CURRENT_STATE.md, and NEXT_TASK.md on its own, then works.
6. When it finishes, it shows you evidence and updates the shared files.
7. You look, you say "yes merge it" or "no, fix X". Done.

## IMPORTANT: do them in ORDER, one at a time (this is the token-safe way)
- Start with **Session 1 (Architect)**. Let it finish and merge BEFORE anything else.
  Everything depends on the foundation. Do not run others in parallel yet.
- After foundation is merged, THEN you can run Backend, Frontend, etc.
- Run the **Reviewer** card whenever a session says "I'm done" — it double-checks the work
  before you have to. This is your safety net so you review less.

## Why one-at-a-time, not seven-at-once?
Because you're solo. Seven live sessions can't see each other and will build things that
don't fit — creating MORE work for you and burning MORE tokens. One well-run session that
calls a Reviewer is calmer, cheaper, and safer. When you hire engineers later, they can each
take a card and run in parallel — the cards already keep them in separate lanes.
